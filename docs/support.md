# Support

The External Integration Module is free and open source. Support is community-based, centred on GitHub.

## Report an issue

Found a bug, or something doesn't work as described? **Create a GitHub issue in the [TrudAX/XppTools](https://github.com/TrudAX/XppTools/issues) repository** — that is the single tracker for all framework code issues.

A good issue includes:

- your D365FO version and the framework model version (commit or download date),
- what you did (setup values, the payload if possible — the [message log](./forms/operations.md#incoming-messages) lets you download it),
- what you expected, and the full error text or infolog.

## Ask a question or propose a feature

Questions and feature ideas are welcome in the same [issue tracker](https://github.com/TrudAX/XppTools/issues). If the question is about a specific tutorial, a comment on the corresponding [blog post](./resources.md) also works.

## Documentation feedback

If a page on this site is wrong or unclear, open an issue in the site's own repository (linked in the footer) — documentation issues are kept separate from framework code issues.

## Contributing

Pull requests to [TrudAX/XppTools](https://github.com/TrudAX/XppTools) are welcome — for example, new connectors, new sample document types, or integrating the export module with Data Management / ER engines. If you have working code to share, open a PR or an issue to discuss it first.
